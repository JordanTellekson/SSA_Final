# SSA Final

A secure software application designed to analyze domains and detect potential phishing threats using cybersecurity indicators such as SSL data, WHOIS information, and domain pattern analysis. The system will provide risk scoring and analysis results to help cybersecurity analysts quickly identify suspicious domains.

# Definition of Done

A User Story or task is considered **Done** when the following conditions are met:

1. The feature or task has been fully implemented according to the acceptance criteria.
2. The code compiles and runs without errors.
3. The code follows agreed-upon coding standards.
4. All changes are committed to a feature branch.
5. A Pull Request has been created and reviewed by at least one team member.
6. The Pull Request has been approved and merged into the `main` branch.
7. The feature has been tested and confirmed to work correctly.
8. The Product Owner has reviewed the result and approved that it meets the requirements.
9. The corresponding GitHub Issue has been moved to **Done** on the Project Board.

---

# Repository Workflow

To maintain code quality and security:

1. Developers create a **feature branch** for each issue.
2. Work is committed and pushed to that branch.
3. A **Pull Request** is opened for review.
4. At least one teammate reviews the changes.
5. After approval, the PR is merged into the protected `main` branch.

Direct pushes to `main` are not allowed (unless given permission by all other teammates).

# Agile Workflow

The team follows a **Scrum-based Agile workflow**:

- Work is organized into **6 weekly sprints**
- Daily **standup updates** are posted in Microsoft Teams
- All work is tracked using **GitHub Issues and Project Boards**
- Each sprint concludes with a **Sprint Demo and review**
